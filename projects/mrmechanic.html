MR. Mechanic

On-demand automotive marketplace API — ~100 endpoints for Pakistan

Laravel 12PHP 8.2MySQLLaravel SanctumFirebase FCMLaravel Breeze

# Overview

MR. Mechanic is the production backend for an on-demand automotive marketplace in Pakistan—connecting drivers with mechanics through geo-tagged service requests, bidding, job lifecycle management, and real-time proximity notifications.

# The challenge

Mechanics needed instant alerts within 10 km without exceeding FCM batch limits. The API required strict validation, OTP rate limiting, and a consistent JSON envelope across ~100 endpoints for mobile clients.

# Solution

Firebase FCM with queued Haversine proximity matching notifies approved mechanics within 10 km (500-token batches, 3 retries). Multi-step KYC onboarding gates marketplace access. Distance-based PKR pricing engine. 14 PHP enums, 41 Form Request validators, OTP rate limiting, and ApexCharts admin analytics.

# Key features

  • ~100 REST API endpoints (Sanctum-authenticated)
  • Geo-tagged requests, mechanic bidding, job lifecycle
  • Offline payment confirmation, dual ratings, disputes
  • FCM proximity alerts (10 km, 500-token batches, 3 retries)
  • Multi-step KYC and distance-based PKR pricing

# Technical highlights

API design

Standardized JSON envelope, resource transformers, and Form Request validation.

Proximity queue

Haversine query → chunked FCM dispatch with retry and dead-letter handling.

KYC workflow

Document upload, admin review states, and mechanic approval gates.

Admin analytics

ApexCharts dashboards for jobs, revenue, and mechanic performance.

# Impact

Production marketplace backend enabling real-time mechanic matching for on-demand automotive services.
